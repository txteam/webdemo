0前言:
主要包括排版规范；数据库表名 命名规范;数据库字段名 命名规范;数据库索引规范：

术语和定义
	规则：必须遵守的原则
	建议：必须加以考虑得原则
	格式：格式规范
	示例：例子
	
数据库表名 命名规范
	表命：	模块名 + "_" + "实体名"
		  	项目名 + "_" + 模块名 + "_" + "实体名"
	表名缩写规则：
	当表名过长时，可对表名中部分单词或全单词进行缩写：
	优先缩写，前缀：模块名，项目名。
	缩写：1、单个长单词可根据实际情况取单个单词前2个字母或3个字母
	
数据库字段名 命名规范
	原则上字段名与实体定义中字段名一致：
	如果表名实体名已经描述清楚实体意义时，
	其中字段名中无需将字段名加上，模块，实体的字符。
	例： oper_organization 中 组织名为 name;
	
数据库索引规范：
	对数据库字段中值区分度不大的字段无需添加索引：例:性别等。
		注：如果查询时需要根据该字段进行查询。可根据实际情况考虑，是否添加联合索引后更为合理。
	对数据库查询中，适度控制对like %x%,in,not null等会造成全表索引的语句
	在查询中，查询条件，尽量不要使用函数处理字段。应考虑将输入值用函数处理为与字段能够对应的值。
